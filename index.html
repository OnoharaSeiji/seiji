Metodologia das apostilas

Cada apostila foi organizada para priorizar retenção, recuperação ativa, aplicação prática e revisão contínua.

Revisão espaçada

Toda aula começa recuperando o conteúdo anterior antes de entrar no novo tema.

Recuperação ativa

As perguntas vêm antes da resposta. O cérebro trabalha para lembrar, não só para reler.

Exemplo guiado + prática

Você entende o conceito, vê uma aplicação e depois executa no seu contexto real.

Intercalação

O cronograma alterna mídia, tracking, automação e comercial para evitar estudo passivo e estreito.

Painel de progresso

Marque cada apostila como concluída para acompanhar seu avanço ao longo das 12 semanas.

Cronograma completo

Cada semana traz quatro dias de conteúdo, uma revisão e um teste prático. Clique em qualquer dia para abrir a apostila correspondente.

Observações de uso

Esta estrutura foi pensada para funcionar direto no GitHub Pages, sem depender de build ou instalação adicional.

Como usar

Suba a pasta inteira para o repositório, habilite o GitHub Pages e abra o index. O progresso fica salvo localmente no navegador.

Como expandir

Você pode editar o arquivo assets/js/study-data.js para alterar temas, títulos e exercícios sem mexer no layout.

Próximo passo recomendado

Depois de publicar a estrutura, a etapa ideal é começar a preencher a apostila 1 com conteúdo mais profundo e visual.
